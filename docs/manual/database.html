    Database

    The Database class provides a wrapper around a Knex.js database connection.

    Overview

    The Database class is used to create a top-level interface to your database.

    Create a new database object using the configuration options described below.

    import Database from '@abw/badger-database';

const mydb = new Database({
  // configuration options
})

    You can also use the database() function which is a wrapper around new Database().

    import { database } from '@abw/badger-database';

const mydb = database({
  // configuration options
})

    Configuration

    Knex Configuration Options

    The configuration parameters should include the client and connection parameters as a minimum, along with any other optional configuration parameters accepted as Knex.js configuration options

    For example, a connection to a sqlite3 database might look like this:

    import Database from '@abw/badger-database'

const database = new Database({
  client: 'sqlite3',
  connection: {
    filename: ':memory:',
  },
  useNullAsDefault: true,
  pool: {
    min: 2,
    max: 10,
  }
})

    queries

    Used to define named SQL queries that you can then run by calling the query(name) method specifying the name of the query.

    const database = new Database(
  // ...client, connection, pool, etc...
  queries: {
    albumsByNumberOfTracks:
      'SELECT albums.title, COUNT(tracks.id) as n_tracks ' +
      'FROM albums ' +
      'JOIN tracks ' +
      'ON tracks.album_id=albums.id ' +
      'GROUP BY albums.id ' +
      'ORDER BY n_tracks ',
  }
)

    See the Queries manual page for further information.

    fragments

    Use to define commonly used SQL fragments that can be interpolated into named queries.

    const database = new Database(
  // ...client, connection, pool, etc...
  fragments: {
    selectAlbumsWithTrackCount:
      'SELECT albums.title, COUNT(tracks.id) as n_tracks ' +
      'FROM albums ' +
      'JOIN tracks ' +
      'ON tracks.album_id=albums.id ' +
      'GROUP BY albums.id '
  },
  queries: {
    albumsByNumberOfTracks:
      ' ' +
      'ORDER BY n_tracks ',
    albumWithMostTracks:
      ' ' +
      'ORDER BY n_tracks DESC ' +
      'LIMIT 1',
  }
)

    See the Queries manual page for further information.

    tables

    Used to define table schemas.

    const database = new Database(
  // ...client, connection, pool, etc...
  tables: {
    users: {
      // ... schema for users table
    },
    companies: {
      // ... schema for companies table
    }
  }
)

    See the Table manual page for further details.

    tablesClass

    This can be used to provide an alternate class for returning table configuration options. The default is the Tables class.

    tablesObject

    This can be used to provide a pre-instantiated object for returning table configuration options. See the Tables pages for further information.

    Methods

    knex()

    Returns a Knex query for the database. Equivalent to calling database.connection.knex().

    const row =
  await database
    .knex('user')
    .select('forename')
    .where({ email: 'bobby@badger.com' })
    .first()

    raw(sql)

    Used to generate a raw SQL query for the database. Equivalent to calling knex.raw().

    const rows =
  await database
    .raw('select forename from user where email="bobby@badger.com"');

    query(name)

    This method allows you to execute a named query that was previously defined using the queries configuration option.

    Given this database definition:

    const database = new Database(
  // ...client, connection, pool, etc...
  queries: {
    albumsByNumberOfTracks:
      'SELECT albums.title, COUNT(tracks.id) as n_tracks ' +
      'FROM albums ' +
      'JOIN tracks ' +
      'ON tracks.album_id=albums.id ' +
      'GROUP BY albums.id ' +
      'ORDER BY n_tracks ',
  }
)

    You can then run the query like so:

    const albums = await database.query('albumsByNumberOfTracks');

    hasTable(name)

    Method to check if a table exists. Returns the configuration options for the table or a false value if the table doesn't exist.

    table(name)

    Method to return a table object for a pre-defined table in the database.

    const users = database.table("users");

    See the Table manual page for further details.

    Functions

    database()

    A function of convenience which wraps a call to new Database().

    import { database } from '@abw/badger-database';
const db = database(...);

    This is equivalent to:

    import { Database } from '@abw/badger-database';
const db = new Database(...);
